vLLMзі»з»ҹжӢҶи§Ј-05-KV CacheдёҺPagedAttentionпјҡж ёеҝғдёҚжҳҜAttentionе…¬ејҸпјҢиҖҢжҳҜеӯҳж”ҫж–№ејҸ

еҰӮжһңиҰҒжүҫ vLLM зӣёжҜ”ж—©жңҹжҺЁзҗҶжЎҶжһ¶жңҖйҮҚиҰҒзҡ„ж”№еҸҳпјҢзӯ”жЎҲйҖҡеёёжҢҮеҗ‘ PagedAttentionгҖӮдҪҶ"PagedAttention"иҝҷдёӘеҗҚеӯ—еҫҲе®№жҳ“иў«иҜҜиҜ»вҖ”вҖ”е®ғ page зҡ„дёҚжҳҜ attention зҡ„и®Ўз®—иҝҮзЁӢпјҢиҖҢжҳҜ attention жүҖдҫқиө–зҡ„ж•°жҚ®зҡ„еӯҳеӮЁж–№ејҸгҖӮ

иҝҷзҜҮж–Үз« д»Һ KV Cache зҡ„еҹәжң¬жңәеҲ¶и®Іиө·пјҢи§ЈйҮҠдёәд»Җд№Ҳиҝһз»ӯеҶ…еӯҳеҲҶй…Қж–№жЎҲеңЁеңЁзәҝжңҚеҠЎдёӯдјҡеӨұж•ҲпјҢvLLM зҡ„ block/page еҢ–и®ҫи®Ўи§ЈеҶідәҶд»Җд№Ҳй—®йўҳпјҢд»ҘеҸҠиҝҷдёӘи®ҫи®ЎеҰӮдҪ•дёҺи°ғеәҰеҷЁгҖҒPrefix Caching еҪўжҲҗзі»з»ҹеҗҲеҠӣгҖӮжңҖеҗҺдјҡеҢәеҲҶ PagedAttention е’Ң FlashAttentionпјҢиҝҷдёӨдёӘжҰӮеҝөеңЁйқўиҜ•йҮҢз»Ҹеёёиў«ж··ж·ҶгҖӮ

KV Cache жҳҜд»Җд№ҲпјҢдёәд»Җд№Ҳеҝ…йЎ»жңүе®ғ

еңЁиҮӘеӣһеҪ’з”ҹжҲҗйҮҢпјҢжЁЎеһӢжҜҸз”ҹжҲҗдёҖдёӘж–° tokenпјҢйғҪйңҖиҰҒеҜ№"д№ӢеүҚжүҖжңү token"еҒҡ attentionгҖӮд»Ҙз”ҹжҲҗз¬¬ 101 дёӘ token дёәдҫӢпјҡ

  • Query еҸӘй’ҲеҜ№еҪ“еүҚиҝҷдёҖжӯҘзҡ„ж–° token
  • Key / Value ж¶үеҸҠеүҚйқўе…ЁйғЁ 100 дёӘеҺҶеҸІ token

еҰӮжһңжҜҸдёҖжӯҘ decode йғҪд»ҺеӨҙжҠҠеҺҶеҸІ token зҡ„ K/V йҮҚж–°з®—дёҖйҒҚпјҢи®Ўз®—йҮҸдјҡйҡҸеәҸеҲ—й•ҝеәҰзәҝжҖ§еўһй•ҝпјҢзҙҜи®Ўд»Јд»·жһҒй«ҳгҖӮжүҖд»Ҙе®һйҷ…зі»з»ҹйғҪдјҡжҠҠеҺҶеҸІ token еҜ№еә”зҡ„ K е’Ң V дҝқеӯҳдёӢжқҘпјҢеҗҺз»ӯ decode ж—¶пјҡ

  1. еҜ№ж–° token еҸӘи®Ўз®—еҪ“еүҚиҝҷдёҖжӯҘзҡ„ Q/K/V
  2. жҠҠж–° K/V иҝҪеҠ иҝӣзј“еӯҳ
  3. attention з”ЁеҪ“еүҚ Q еҺ»иҜ»еҺҶеҸІзј“еӯҳдёӯзҡ„ K/V

иҝҷдёӘдҝқеӯҳдёӢжқҘзҡ„еҺҶеҸІзҠ¶жҖҒе°ұжҳҜ KV CacheгҖӮ

иҝҷдёҚжҳҜ"й”ҰдёҠж·»иҠұ"зҡ„дјҳеҢ–пјҢиҖҢжҳҜзҺ°д»Ј LLM жңҚеҠЎдёӯеҮ д№Һй»ҳи®ӨеӯҳеңЁзҡ„еҹәзЎҖжңәеҲ¶гҖӮжІЎжңүе®ғпјҢй•ҝеәҸеҲ—з”ҹжҲҗзҡ„д»Јд»·дјҡйҡҸжӯҘж•°е№іж–№зә§еўһй•ҝгҖӮ

дёәд»Җд№Ҳ KV Cache дјҡжҲҗдёәеңЁзәҝжңҚеҠЎзҡ„дё»иҰҒз“¶йўҲ

жЁЎеһӢжқғйҮҚжҳҜеӣәе®ҡзҡ„пјҢдҪҶ KV Cache жҳҜеҠЁжҖҒеўһй•ҝзҡ„гҖӮиҜ·жұӮи¶Ҡй•ҝгҖҒз”ҹжҲҗи¶Ҡд№…пјҢKV Cache и¶ҠеӨ§пјӣиҖҢдё”дёҚжҳҜдёҖдёӘиҜ·жұӮеңЁж¶ЁпјҢжҳҜжүҖжңүе№¶еҸ‘иҜ·жұӮйғҪеңЁж¶ЁгҖӮ

KV Cache еҝ…йЎ»й©»з•ҷеңЁ GPU дёҠпјҢеӣ дёә decode жҜҸдёҖжӯҘйғҪиҰҒй«ҳйў‘иҜ»еҸ–е®ғгҖӮе®ғе’ҢжЁЎеһӢжқғйҮҚгҖҒжҝҖжҙ»еҖјгҖҒдёӯй—ҙ buffer дёҖиө·дәүеӨәжҳҫеӯҳгҖӮеңЁеҫҲеӨҡеңЁзәҝжңҚеҠЎеңәжҷҜйҮҢпјҢзңҹжӯЈеҚЎдҪҸ batch size зҡ„пјҢдёҚжҳҜз®—еҠӣпјҢиҖҢжҳҜ KV Cache зҡ„жҳҫеӯҳеҚ з”ЁдёҺз®ЎзҗҶж–№ејҸгҖӮ

иҝҷе’Ң decode йҳ¶ж®өзҡ„и®Ўз®—зү№жҖ§еҜҶеҲҮзӣёе…ігҖӮDecode жҳҜ memory-bound зҡ„пјҡжҜҸдёҖжӯҘж–°з®—зҡ„дёңиҘҝдёҚеӨҡпјҲеҸӘжңүдёҖдёӘ tokenпјүпјҢдҪҶиҰҒйў‘з№Ғи®ҝй—®еҺҶеҸІ K/VгҖӮжүҖд»Ҙ KV Cache зҡ„еӯҳеӮЁз»“жһ„е’Ңи®ҝй—®ж–№ејҸпјҢеҜ№ decode е»¶иҝҹзҡ„еҪұе“ҚжҜ”еҜ№ prefill жӣҙзӣҙжҺҘгҖӮvLLM е®ҳж–№дјҳеҢ–ж–ҮжЎЈжҠҠ decode жҳҺзЎ®жҸҸиҝ°дёә memory-boundпјҢиҝҷд№ҹжҳҜ vLLM жҠҠеӨ§йҮҸи®ҫи®ЎйҮҚеҝғж”ҫеңЁ KV Cache з®ЎзҗҶдёҠзҡ„еҺҹеӣ гҖӮ

иҝһз»ӯеҲҶй…Қдёәд»Җд№ҲдёҚйҖӮеҗҲеңЁзәҝжңҚеҠЎ

зӣҙи§үдёҠпјҢжңҖз®ҖеҚ•зҡ„еҒҡжі•жҳҜз»ҷжҜҸдёӘиҜ·жұӮеҲҶй…ҚдёҖж®өиҝһз»ӯжҳҫеӯҳжқҘеӯҳ KV CacheпјҢеәҸеҲ—еҸҳй•ҝж—¶з»§з»ӯеҫҖеҗҺжү©гҖӮиҝҷдёӘж–№жЎҲеңЁзҰ»зәҝжү№йҮҸжҺЁзҗҶйҮҢеҸҜд»Ҙе·ҘдҪңпјҢдҪҶеңЁй«ҳе№¶еҸ‘еңЁзәҝжңҚеҠЎйҮҢдјҡжҡҙйңІеӣӣдёӘж №жң¬й—®йўҳгҖӮ

й•ҝеәҰдёҚеҸҜйў„зҹҘгҖӮ иҜ·жұӮиҝӣжқҘж—¶пјҢдҪ дёҚзҹҘйҒ“е®ғжңҖз»Ҳдјҡз”ҹжҲҗеӨҡй•ҝгҖӮдҝқе®ҲеҲҶй…ҚпјҲе…ҲеҲҶе°ҸпјҢдёҚеӨҹеҶҚжү©пјүдјҡеёҰжқҘйў‘з№Ғжү©е®№е’Ңж•°жҚ®жҗ¬иҝҒпјӣжҝҖиҝӣйў„з•ҷпјҲдёҖејҖе§ӢеӨҡйў„з•ҷпјүдјҡеӨ§йҮҸжөӘиҙ№жҳҫеӯҳгҖӮиҝҷдёӘзҹӣзӣҫж— жі•йҖҡиҝҮи°ғеҸӮж №жң¬и§ЈеҶігҖӮ

еӨ–йғЁзўҺзүҮгҖӮ дёҚеҗҢиҜ·жұӮзҡ„з”ҹе‘Ҫе‘Ёжңҹе·®ејӮжһҒеӨ§гҖӮзҹӯиҜ·жұӮеҝ«йҖҹз»“жқҹпјҢйҮҠж”ҫдәҶдёҖж®өдёӯй—ҙз©әй—ҙпјӣж–°иҜ·жұӮйңҖиҰҒжӣҙеӨ§зҡ„иҝһз»ӯеҢәпјҢдҪҶиҝҷж®өз©әжҙһдёҚеӨҹз”ЁгҖӮжҖ»з©әй—Іжҳҫеӯҳи¶іеӨҹпјҢеҚҙеӣ дёәдёҚиҝһз»ӯиҖҢж— жі•еҲ©з”ЁвҖ”вҖ”иҝҷжҳҜз»Ҹе…ёзҡ„еӨ–йғЁзўҺзүҮй—®йўҳгҖӮ

жү©е®№йңҖиҰҒжҗ¬иҝҒгҖӮ жҹҗдёӘиҜ·жұӮзҡ„ KV Cache з»§з»ӯеўһй•ҝпјҢдҪҶеҺҹжқҘйӮЈж®өиҝһз»ӯеҢәеҗҺйқўе·Іиў«еҚ з”ЁгҖӮзі»з»ҹиҰҒжүҫдёҖеқ—жӣҙеӨ§зҡ„иҝһз»ӯеҢәпјҢжҠҠжүҖжңү K/V еӨҚеҲ¶иҝҮеҺ»пјҢеҶҚжӣҙж–°еј•з”ЁгҖӮеңЁй«ҳе№¶еҸ‘еңәжҷҜдёӢпјҢиҝҷдёӘд»Јд»·йқһеёёй«ҳгҖӮ

еүҚзјҖе…ұдә«дёҚиҮӘз„¶гҖӮ еҰӮжһңдёӨдёӘиҜ·жұӮжңүзӣёеҗҢзҡ„ system promptпјҢзҗҶжғіжғ…еҶөдёӢиҝҷйғЁеҲҶ K/V еҸҜд»Ҙе…ұз”ЁгҖӮдҪҶеңЁиҝһз»ӯеӨ§ tensor жЁЎеһӢдёӢпјҢеҫҲйҡҫжҠҠ"е…ұдә«зҡ„еҸӘиҜ»еүҚзјҖ"е’Ң"еҗ„иҮӘж–°еўһзҡ„йғЁеҲҶ"еҲҶејҖз®ЎзҗҶпјҢйҖҡеёёзҡ„з»“жһңжҳҜеҗ„иҮӘдҝқз•ҷдёҖд»ҪпјҢз”ҡиҮійҮҚж–°и®Ўз®—дёҖд»ҪгҖӮ

vLLM зҡ„и§Јжі•пјҡжҠҠ KV Cache еҲҮжҲҗеӣәе®ҡеӨ§е°Ҹзҡ„ block

vLLM дёҚз»ҷжҜҸдёӘиҜ·жұӮеҮҶеӨҮдёҖж®өиҝһз»ӯжҳҫеӯҳпјҢиҖҢжҳҜпјҡ

  • жҠҠ KV Cache еҲҮжҲҗеҫҲеӨҡеӣәе®ҡеӨ§е°Ҹзҡ„ blockпјҲд№ҹеҸ« pageпјү
  • жҜҸдёӘ block е®№зәіеӣәе®ҡж•°йҮҸ token зҡ„ K/V
  • дёҖдёӘиҜ·жұӮзҡ„йҖ»иҫ‘еәҸеҲ—з”ұеӨҡдёӘ block дёІиө·жқҘ
  • иҝҷдәӣ block еңЁзү©зҗҶжҳҫеӯҳдёҠдёҚйңҖиҰҒиҝһз»ӯ

иҝҷдёӘжҖқи·ҜзӣҙжҺҘжқҘиҮӘж“ҚдҪңзі»з»ҹзҡ„еҲҶйЎөеҶ…еӯҳз®ЎзҗҶпјҡ

д»Ҙ block size = 16 дёәдҫӢпјҢдёҖдёӘиҜ·жұӮзҡ„еҺҶеҸІ token еңЁйҖ»иҫ‘дёҠжҳҜпјҡ

дҪҶзү©зҗҶдёҠеҸҜиғҪжҳҜпјҡ

еҜ№иҜ·жұӮжқҘиҜҙпјҢеҺҶеҸІд»Қз„¶жҳҜиҝһз»ӯзҡ„пјӣеҜ№зі»з»ҹжқҘиҜҙпјҢеҸӘйңҖз»ҙжҠӨдёҖд»Ҫ"йҖ»иҫ‘еқ—зј–еҸ· вҶ’ зү©зҗҶеқ—дҪҚзҪ®"зҡ„жҳ е°„гҖӮ

"paged"еҲ°еә• page дәҶд»Җд№Ҳ

иҝҷжҳҜжңҖе®№жҳ“дә§з”ҹж··ж·Ҷзҡ„ең°ж–№гҖӮ

PagedAttention дёӯпјҢиў« page еҢ–зҡ„жҳҜ KV Cache зҡ„еӯҳеӮЁдёҺзҙўеј•ж–№ејҸпјҢдёҚжҳҜ attention зҡ„и®Ўз®—ж•°еӯҰгҖӮ

attention д»Қз„¶еңЁеҒҡпјҡеҪ“еүҚ query еҜ№еҺҶеҸІ keys еҒҡзӣёе…іжҖ§и®Ўз®—пјҢз”ЁжқғйҮҚеҠ жқғеҺҶеҸІ valuesгҖӮдёҚеҗҢзҡ„жҳҜпјҢеҺҶеҸІ K/V дёҚеҶҚеҒҮи®ҫеӯҳеңЁдёҖж®өиҝһз»ӯеҶ…еӯҳйҮҢпјҢиҖҢжҳҜйҖҡиҝҮ block table д»ҺеӨҡдёӘеҲҶж•Јзҡ„ block дёӯеҸ–еҮәжқҘеҸӮдёҺи®Ўз®—гҖӮ

еҸҳеҢ–зҡ„жҳҜпјҡ

  • ж•°жҚ®з»„з»Үж–№ејҸ
  • еҶ…еӯҳз®ЎзҗҶж–№ејҸ
  • attention kernel зҡ„и®ҝеӯҳйҖ»иҫ‘

дёҚеҸҳзҡ„жҳҜпјҡattention зҡ„ж•°еӯҰжң¬иә«гҖӮ

еҪ“еҺҶеҸІ K/V жҳҜиҝһз»ӯеӯҳеӮЁж—¶пјҢkernel еҸҜд»ҘзӣҙжҺҘжҢүеҒҸз§»йҮҸеҜ»еқҖгҖӮеҲҮжҚўеҲ° paged layout еҗҺпјҢkernel еңЁеӨ„зҗҶжҜҸдёӘ block ж—¶пјҢйңҖиҰҒе…ҲжҹҘ block table зЎ®е®ҡзү©зҗҶдҪҚзҪ®пјҢеҶҚеҠ иҪҪеҜ№еә”ж•°жҚ®гҖӮvLLM дҪҝз”Ёдё“й—Ёдёә paged KV Cache и®ҫи®Ўзҡ„ multi-head query attention kernelвҖ”вҖ”е®ҳж–№ Paged Attention и®ҫи®Ўж–ҮжЎЈд№ҹзү№еҲ«иҜҙжҳҺпјҢиҝҷйҮҢзҡ„"block"жҳҜ KV cache blockпјҢдёҚжҳҜ GPU thread blockпјҢдёӨиҖ…дёҚиҰҒж··ж·ҶгҖӮ

зҗҶи§ЈиҝҷдёӘи®ҫи®Ўзҡ„е…ій”®дёҖеҸҘиҜқпјҡйҖ»иҫ‘иҝһз»ӯпјҢзү©зҗҶзҰ»ж•ЈпјҢи®Ўз®—ж—¶жҢүжҳ е°„еҸ–ж•°гҖӮ

Block еҢ–и®ҫи®Ўи§ЈеҶідәҶе“Әдәӣй—®йўҳ

еҺҹжңүй—®йўҳ block еҢ–еҰӮдҪ•и§ЈеҶі
й•ҝеәҰдёҚеҸҜйў„зҹҘпјҢйҡҫд»Ҙйў„з•ҷ жҢүйңҖиҝҪеҠ ж–° blockпјҢж— йңҖеҲқе§ӢеҢ–ж—¶зҢңжөӢжңҖз»Ҳй•ҝеәҰ
еӨ–йғЁзўҺзүҮ еҲҶй…ҚеҚ•дҪҚеҸҳдёәеӣәе®ҡеӨ§е°Ҹ blockпјҢеӣһж”¶еҗҺеҸҜзӣҙжҺҘз»ҷж–°иҜ·жұӮеӨҚз”Ё
жү©е®№йңҖиҰҒжҗ¬иҝҒ еўһй•ҝеҸӘйңҖиҝҪеҠ ж–° blockпјҢе·Іжңү block еҺҹдҪҚдёҚеҠЁ
еүҚзјҖе…ұдә«дёҚиҮӘз„¶ еүҚзјҖеҜ№еә”зҡ„ full block еҸҜзӣҙжҺҘе…ұз”ЁпјҢеҲҶеҸүеҗҺеҗ„иҮӘиҝҪеҠ ж–° block
е№¶еҸ‘жүҝиҪҪйҮҸеҸ—йҷҗ зўҺзүҮеҮҸе°‘гҖҒеӣһж”¶й«ҳж•ҲпјҢеҗҢзӯүжҳҫеӯҳдёӢиғҪе®№зәіжӣҙеӨҡжҙ»и·ғеәҸеҲ—

еүҚзјҖе…ұдә«иҝҷдёҖзӮ№еҖјеҫ—еұ•ејҖгҖӮblock еҢ–д№ӢеҗҺпјҢеүҚзјҖеӨҚз”ЁеҸҳжҲҗпјҡ

е…ұдә«еүҚзјҖзӯүдәҺе…ұз”Ёж—§ blockпјҢеҲҶеҸүеҗҺеҗ„иҮӘжҢӮж–° blockгҖӮеҰӮжһңеә•еұӮд»ҚжҳҜиҝһз»ӯеӨ§ tensorпјҢиҝҷз§Қз»ҶзІ’еәҰе…ұдә«еҮ д№Һж— жі•е®һзҺ°гҖӮ

Block Size зҡ„жқғиЎЎ

ж—ўз„¶еҲҮжҲҗ blockпјҢblock еӨ§е°Ҹе°ұжҳҜдёҖдёӘи®ҫи®ЎеҸӮж•°пјҡ

block еӨӘе°Ҹ block еӨӘеӨ§
дјҳзӮ№ еҲҶй…ҚзІ’еәҰз»ҶпјҢе°ҫйғЁжөӘиҙ№е°‘пјӣеүҚзјҖеӨҚз”ЁжӣҙзҒөжҙ» е…ғж•°жҚ®е°‘пјҢжҳ е°„е…ізі»з®ҖеҚ•пјӣkernel и®ҝеӯҳжӣҙи§„ж•ҙ
зјәзӮ№ block ж•°йҮҸеӨҡпјҢе…ғж•°жҚ®з®ЎзҗҶеӨҚжқӮпјӣkernel зҙўеј•ејҖй”ҖеўһеҠ  е°ҫйғЁжңӘз”Ёз©әй—ҙеўһеӨҡпјӣз»ҶзІ’еәҰеӨҚз”ЁиғҪеҠӣеҸҳејұ

жң¬иҙЁжҳҜеңЁз®ЎзҗҶзІ’еәҰгҖҒжҳҫеӯҳжөӘиҙ№гҖҒзҙўеј•еӨҚжқӮеәҰгҖҒkernel е®һзҺ°йҡҫеәҰеӣӣиҖ…д№Ӣй—ҙеҸ–е№іиЎЎгҖӮе…·дҪ“й»ҳи®ӨеҖјдјҡйҡҸеҗҺз«Ҝе®һзҺ°е’ҢжЁЎеһӢй…ҚзҪ®еҸҳеҢ–пјҢжІЎжңүдёҖдёӘйҖҡз”ЁжңҖдјҳзӯ”жЎҲгҖӮ

Prefix Caching дёәд»Җд№Ҳе’Ң paged KV Cache еӨ©з„¶жҗӯжЎЈ

vLLM зҡ„ Prefix Caching д»Ҙ block дёәзІ’еәҰзј“еӯҳе·Іи®Ўз®—зҡ„ KVпјҡ

  • еҸӘжңү full blockпјҲе·Іиў«еЎ«ж»Ўзҡ„ blockпјүдјҡиҝӣе…Ҙзј“еӯҳ
  • жҜҸдёӘ full block зҡ„зј“еӯҳй”®з”ұд»ҘдёӢеҶ…е®№иҒ”еҗҲе“ҲеёҢеҫ—еҮәпјҡзҲ¶ block зҡ„ hashгҖҒеҪ“еүҚ block зҡ„ token еәҸеҲ—пјҢд»ҘеҸҠ LoRA IDгҖҒеӨҡжЁЎжҖҒиҫ“е…Ҙ hashгҖҒcache salt зӯүйҷ„еҠ дҝЎжҒҜ
  • ж–°иҜ·жұӮе‘Ҫдёӯж—¶пјҢжҢү block йҖҗдёӘеҢ№й…ҚпјҢзӣҙеҲ°жҹҗдёӘдҪҚзҪ®ејҖе§ӢдёҚеҗҢдёәжӯўпјҢеҗҺз»ӯеҶҚж–°е»ә block

еҰӮжһңеә•еұӮжҳҜиҝһз»ӯ tensorпјҢвҖңзј“еӯҳйғЁеҲҶеүҚзјҖ"еңЁж•°жҚ®з»“жһ„дёҠдјҡжӣҙз¬ЁйҮҚпјӣblock еҢ–д№ӢеҗҺпјҢprefix caching еҮ д№ҺжҳҜ block еҢ–и®ҫи®Ўзҡ„иҮӘз„¶е»¶дјёвҖ”вҖ”еүҚзјҖеӨҚз”Ёе°ұжҳҜ"е‘ҪдёӯдёҖжү№ж—§ blockвҖқгҖӮ

дёәд»Җд№Ҳ KV Cache з®ЎзҗҶе’Ңи°ғеәҰеҷЁеҝ…йЎ»еҚҸеҗҢ

KV Cache дёҚжҳҜзӢ¬з«ӢжЁЎеқ—пјҢе®ғе’Ңи°ғеәҰеҷЁжҳҜзҙ§иҖҰеҗҲзҡ„пјҲз¬¬ 4 зҜҮдёӯе·Ід»Ӣз»Қи°ғеәҰеҷЁзҡ„е·ҘдҪңжңәеҲ¶пјүгҖӮ

и°ғеәҰеҷЁжҜҸдёҖиҪ®зҡ„еҶізӯ–йғҪдҫқиө– KV Cache зҡ„еҪ“еүҚзҠ¶жҖҒпјҡиҝҳеү©еӨҡе°‘ free blockгҖҒе“Әдәӣ block иў«е“ӘдәӣиҜ·жұӮеј•з”ЁгҖҒе“Әдәӣ cached block еҸҜд»ҘеӨҚз”ЁгҖӮеҸҚиҝҮжқҘпјҢи°ғеәҰеҷЁзҡ„еҶізӯ–д№ҹзӣҙжҺҘеҪұе“Қ KV Cache зҡ„еҲҶй…ҚдёҺеӣһж”¶иҠӮеҘҸгҖӮ

иҝҷд№ҹжҳҜ vLLM жһ¶жһ„дёӯ scheduler е’Ң KV cache manager иў«з»ҹдёҖж”ҫеңЁ engine core йҮҢдёҖдҪ“еҢ–жҖқиҖғзҡ„еҺҹеӣ гҖӮ

PagedAttention зҡ„ж”¶зӣҠпјҡдёҚеҸӘжҳҜзңҒжҳҫеӯҳ

жҠҠ PagedAttention з®ҖеҚ•зҗҶи§ЈжҲҗ"жҳҫеӯҳдјҳеҢ–"жІЎжңүй”ҷпјҢдҪҶеҸӘзңӢеҲ°дәҶз¬¬дёҖеұӮгҖӮе®Ңж•ҙзҡ„ж”¶зӣҠй“ҫжқЎжҳҜпјҡ

жҳҫеӯҳзўҺзүҮеҮҸе°‘гҖҒжҢүйңҖеўһй•ҝ

вҶ“ еҗҢзӯүжҳҫеӯҳдёӢиғҪе®№зәіжӣҙеӨҡжҙ»и·ғеәҸеҲ—

вҶ“ и°ғеәҰеҷЁжүӢйҮҢеҸҜз”ЁеҖҷйҖүиҜ·жұӮжӣҙеӨҡпјҢcontinuous batching жӣҙжңүж•Ҳ

вҶ“ Prefix caching зҡ„еӨҚз”ЁзІ’еәҰжӣҙз»ҶпјҢе‘Ҫдёӯж”¶зӣҠжӣҙеӨ§

вҶ“ ж•ҙдҪ“ serving еҗһеҗҗжӣҙй«ҳпјҢе»¶иҝҹжіўеҠЁжӣҙе°Ҹ

vLLM и®әж–ҮпјҲEfficient Memory Management for Large Language Model Serving with PagedAttentionпјүе°ҶеҗһеҗҗжҸҗеҚҮе’Ң near-zero waste ж”ҫеңЁдёҖиө·и®ІпјҢе°ұжҳҜеңЁејәи°ғпјҡжҳҫеӯҳз®ЎзҗҶдёҚжҳҜйҷ„еұһдјҳеҢ–пјҢиҖҢжҳҜзӣҙжҺҘеҪұе“Қе№¶еҸ‘иғҪеҠӣе’ҢеҗһеҗҗиғҪеҠӣзҡ„ж ёеҝғеӣ зҙ гҖӮ

иҝҷз§Қи®ҫи®Ўзҡ„д»Јд»·

е®һзҺ°еӨҚжқӮеәҰй«ҳгҖӮ иҝһз»ӯ tensor зҡ„жҖқз»ҙеҫҲзӣҙи§Ӯпјӣpaged KV Cache йңҖиҰҒ block з®ЎзҗҶгҖҒblock tableгҖҒеј•з”Ёи®Ўж•°гҖҒеӣһж”¶зӯ–з•ҘгҖҒдёҺи°ғеәҰеҷЁеҚҸи°ғгҖҒе®ҡеҲ¶еҢ– attention kernelгҖӮе·ҘзЁӢеӨҚжқӮеәҰжҳҺжҳҫжӣҙй«ҳгҖӮ

Kernel ж— жі•еҒҮи®ҫеҶ…еӯҳиҝһз»ӯгҖӮ еӣ жӯӨйңҖиҰҒдё“й—ЁйҖӮй…Қ paged layout зҡ„ attention kernelпјҢдёҚиғҪзӣҙжҺҘеӨҚз”ЁеҺҹжңүзҡ„иҝһз»ӯеҶ…еӯҳе®һзҺ°гҖӮе®ҳж–№и®ҫи®Ўж–ҮжЎЈдё“й—ЁдёәжӯӨзӢ¬з«ӢжҲҗйЎөпјҢиҜҙжҳҺиҝҷдёҚжҳҜдёҖдёӘе°Ҹж”№еҠЁгҖӮ

е…ғж•°жҚ®з®ЎзҗҶжңүејҖй”ҖгҖӮ block tableгҖҒhash и®Ўз®—гҖҒеј•з”Ёи®Ўж•°гҖҒfree block queue йғҪдёҚжҳҜйӣ¶жҲҗжң¬гҖӮеңЁеӨ§и§„жЁЎеңЁзәҝжңҚеҠЎеңәжҷҜдёӢпјҢиҝҷдәӣејҖй”ҖзӣёеҜ№ж”¶зӣҠйҖҡеёёеҖјеҫ—пјӣдҪҶеңЁе°Ҹи§„жЁЎжҲ–еҚ•иҜ·жұӮеңәжҷҜдёӢпјҢжҜ”еҖјдёҚдёҖе®ҡеҲ’з®—гҖӮ

йқһж ҮеҮҶзЎ¬д»¶зҡ„йҖӮй…ҚйҡҫеәҰжӣҙеӨ§гҖӮ еӣ дёәж¶үеҸҠеә•еұӮи®ҝеӯҳе’Ң kernel и®ҫи®ЎпјҢжҹҗдәӣ NPU жҲ–иҫ№зјҳи®ҫеӨҮиҰҒе®Ңж•ҙеӨҚзҺ° vLLM зҡ„жҖ§иғҪзү№жҖ§пјҢйҡҫзӮ№еҫҖеҫҖйӣҶдёӯеңЁ paged attention иҝҷдёҖеұӮгҖӮ
